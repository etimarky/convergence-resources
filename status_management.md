## Status Management

Statuses are used to track the progress of categories, attributes, and items. This includes tracking approvals, rejections, and comments.

Managing these statuses is done in Catalog Management.&#x20;

&#x20;

Permissions need to be set and allowed for users to utilize the catalog manager and status manager.&#x20;

These permissions include:&#x20;

-Classification Permission: Category Permission/ Edit Items

-Catalog Permission: Access Catalog Status Module  Locate the Catalog Management tab under Administration.&#x20;

&#x20;

![](.gitbook/assets/image\(893\).png)

&#x20;

Select Status Management

&#x20;

![](.gitbook/assets/image\(894\).png)

&#x20;

The next screen will show all of the available statuses.&#x20;

Users can create as many statuses as they would like. Each status is identified by name and a color that the user determines themselves.&#x20;

&#x20;

New statuses can only be created between "New" and "Pending".&#x20;

Statuses such as New, Pending, Approved, and Rejected are all standard system statuses and cannot be edited or removed.&#x20;

&#x20;

Check boxes are available for Categories, Attributes, and Parts.&#x20;

One or multiple boxes can be checked on each status: this means the status will apply to that selection.&#x20;

For example:  A category, and part can both be selected to go through "New, In Work, and Classification" wheras an attribute may only need to go through "New and In Work."  &#x20;
