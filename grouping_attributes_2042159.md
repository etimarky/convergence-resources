**Grouping Attributes**


 


Groups allows several attributes to be associated together. 


 


Grouping attributes has several advantages:


- It encourages consistency in the way similar attributes are created;
- Grouping similar or related attributes makes it easier to assign the correct attribute to a category. For example, a Material group could contain all attributes related to material including MATERIAL GRADE, MATERIAL THICKNESS, MATERIAL SPECIFICATION, MATERIAL HARDNESS, etc.
- Groups can be used by the filter to sort through the larger Master Attribute List to produce a shorter, more specific list of attributes.
- As items are added to the classification, all applicable attributes can be displayed in and selected from a single group.

 


Groups can also be used to create a list of attributes associated with an item collection. To learn about creating and using groups in item collections, reference the **Item Collections** [Define a Group of Attributes about a Relationship](hs8200.md) topic.


 


The **Group** tab at the bottom right displays the groups to which the current attribute is assigned. To [add an attribute to a group](javascript:popuplink('hs2320.htm')), the group name is added to the list. To [remove an attribute from a group](javascript:popuplink('hs2330.htm')), the group name is deleted from list.


 


Pressing the **Group** tab activates the [Define Group](javascript:popuplink('.htm')) function which allows the user to define a new group. 


 


The user may also retrieve a list of attributes assigned to an existing group using the [Attribute Filter](javascript:popuplink('hs2400.htm')).
