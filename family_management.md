# Family Management


Family Management allows the user to create product families and segments that will form the product hagiarchy. 


A product family must be added on this screen, otherwise the user will not be able to load segment data against it. 


 


 


Inside the SmartTeardown module, select Family Management. 
 


![](assets/image(52).png)




 


Once inside Family Management, the user has the option to add, edit, delete or export a family. 


To add a family, navigate to the left half of the screen and select the orange icon. This will allow the user to then select "Add." 


 


![](assets/image(54).png)




 


Fill in all pertinent information such as Name and Description.


The user can also add an image at any time, whether that's upon initial creation or while going back and editing at a later time. 


 


![](assets/image(57).png)




 


On the right side of the screen, families are able to be edited, deleted or exported. 


Regions, feature sets and product structure are also built out and determined on this screen as well. 


 


![](assets/image(58).png)




 


Regions are defined in General Management and once a region is selected, a product with the selected region(s) will only show up when that region is chosen in the drop down on the top right of the screen. 


If no regions are selected, all regions are assumed.


 


![](assets/image(59).png)




 


Feature sets are set up on this screen as well. Simply type in a feature set and select enter to add it. These can then be selected on products when creating on the SmartTeardown main screen. 


Feature sets can also be edited and deleted on this screen as well. 


 


![](assets/image(61).png)




 


Lastly, Product structure is built out on this screen. 


This is similar to building out structure in SmartClass. Define classes/categories and build them out. 


There can be no more than 5 levels in the product structure. 


 


Product structure can also be deleted and edited on this screen as well. 


If a user deletes product structure that has segment data attached to it, the data will be lost indefinitely.   

There will be one warning notifying the user of this. The deletion cannot be undone. 
 


![](assets/image(65).png)




 


Product Structure once built out can be exported, which will automatically create a template that can be edited and later imported back in to include segment data. 


Below is an example of an export of dryers.   

 


When filling out segment data information, Region, Materials and Manufacturer will appear as drop-down options which easily allow the user to select these options based on information set up in General Management. The user will not be able to freely type in information in these areas. 
Model must be filled in as well but will not appear in a drop down.
Fasteners and Segment data tabs are important in this template but "ListInfo" is just that- a list and can be ignored when setting up segment and fastener data.
