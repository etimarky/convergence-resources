**Printing 
Road Maps and Summary Metrics**



Users will be able to print road map data when selecting the 
print option from the File menu.  You will also notice that the bottom of the 
road map will contain summary information on the roadmap including:


- # of modules
- # of module variants
- Ration of variants to modules
- Cost summaries, etc.

## Module Level Roadmaps


The module level roadmap summarizes the variant data in a 
single row.  The end of a row in this road represents the final variant year.  
Some rows may end at another row since the last variant may be replaced by a 
variant in another module.  The second row in the figure below describes that 
condition.


 


![](/userfiles/13115/16830/ckfinder/images/import-d2f34nel28gg/IM-Print_Summary_files/image003.jpg)




**Example of a Module Level Roadmap**



## Module Summary Report


The Module Type Summary report provides a rollup of all of 
the module, variant, EAU and cost forecast data across all module types in the 
database.  This report can be run while Item Mapping is open from the Reports 
menu as described below.


 


![](/userfiles/13115/16830/ckfinder/images/import-d2f34nel28gg/IM-Print_Summary_files/image004.jpg)




**Example of selecting Module Type Summary report in DFR**


 


Users will need access to a printer that can support the 11 
X 17 format.  When you run the Module Type Summary report, a dialog box will 
present all of your 11 X 17 printing options including applications like 
Adobe.  The application, temporarily will set your default printer to the 
printer you select that supports 11 X 17.  


![](/userfiles/13115/16830/ckfinder/images/import-d2f34nel28gg/IM-Print_Summary_files/image005.jpg)




**Example of how the printing works for 11X17 formats.**


 


Once you have completed your printing and closed the Module 
Type Summary report, your default printer will return to its previous default 
printer setting.


 


Users will also need to determine what owner groups to 
generate a summary report against.  The owner group list is configured to show 
the following owner groups first, see below.


 


![image001](/userfiles/13115/16830/ckfinder/images/import-d2f34nel28gg/IM-Print_Summary_files/image006.gif)




 


**Example of Owner Group Mappings**



 


![](/userfiles/13115/16830/ckfinder/images/import-d2f34nel28gg/IM-Print_Summary_files/image007.jpg)




**Example of Owner Group order in Item Mapping**
