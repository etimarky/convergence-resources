**Using Allowed Values Lists**


 


Once data is loaded, the Data Developer tool allows a user to view an entire batch of data and each item's attribute values. In a spreadsheet format, individual item's attribute values can be modified one at a time, or using sorting functions, in small groups. However, the Data Developer relies on the user's ability to know all the possible values of a specific attribute and his ability to find and replace those values with standard values. 


 


The Allowed Values Manager is a powerful tool used to target individual attributes and define a list of specific, standard string values that can apply to the attribute. The user selects a category and an attribute, and creates a list of possible values for that attribute in that category. Once created, the Allowed Values List is made available to the Data Developer Module to insure that attribute values in the database are restricted to the allowed values.


 


To learn more about Allowed Values Lists, review the [Allowed Value Manager](hs7000.md) topic.


 


The Allowed Values Lists become available in the Data Development Module when the "Restrict to Allowed Values" box is checked at the top of the Data Developer Module.
