# How to Import New Classification Data

## How to Import New Classification Data

Classification data consists of structure: categories, attributes, and mapping the two. LOVs can also be mapped during this initial import.&#x20;

The classification data import is done before item/part data can be loaded and mapped to categories.&#x20;

&#x20;

&#x20;

&#x20;

&#x20;

Select Classification import in the Import Manager

&#x20;

![](.gitbook/assets/image\(1156\).png)

&#x20;

For an example on how to set up a classification spreadsheet, [click here](how_to_import_new_classification_data.md).

Browse for the file on your computer

&#x20;

![](.gitbook/assets/image\(1157\).png)

&#x20;

Select Excel as the Import Format

&#x20;

![](.gitbook/assets/image\(1158\).png)

&#x20;

Depending on how your spreadsheet is set up, select what information you will be uploading.&#x20;

&#x20;

![](.gitbook/assets/image\(1159\).png)

The mapping windows will come next.&#x20;

Depending on the spreadsheet setup, different windows will appear, but will all essentially be set up the same way.&#x20;

&#x20;

### Attribute Mappings:

The current classification import spreadsheet has attributes that are not already in the system. They will need to be created as new.&#x20;

The operation for this would be "create."&#x20;

&#x20;

If attributes do exist in the system, or if some exist and others don't, select " Create or Ignore," This way, the attributes that do not exist will be created, but Convergence PIM will not try to duplicate existing attributes.&#x20;

Update operation currently not supported. Available in future release.

Create or Update operation currently not supported. Available in future release.

Update or Ignore operation currently not supported. Available in future release.

&#x20;

If the spreadsheet is set up correctly, all mapping fields will be auto-mapped. If they are not, simply select to the right of "Clear Mapping" and select the proper column heading name.

&#x20;

![](.gitbook/assets/image\(1160\).png)

&#x20;

### Category Mapping

Category Mapping exists and functions the same way as the attribute mapping does.&#x20;

The operation for this spreadsheet will be create, as the categories are brand new along with the new attributes above.&#x20;

&#x20;

If it's new structure going in, select either "Create" or "Create or Ignore" to direct Convergence PIM to create new categories.&#x20;

Update operation currently not supported. Available in future release.

Create or Update operation currently not supported. Available in future release.

Update or Ignore operation currently not supported. Available in future release.

&#x20;

![](.gitbook/assets/image\(1161\).png)

&#x20;

### Category Attribute Mapping

Lastly is the category attribute mapping screen which is the screen that tells Convergence PIM what attributes are being applied to which categories.&#x20;

The operation for this spreadsheet is "Create," as the mappings are brand new, along with the new attributes and new categories above.&#x20;

&#x20;

"Create" or "Create or Ignore" for new mappings.&#x20;

Update operation currently not supported. Available in future release.

Create or Update operation currently not supported. Available in future release.

Update or Ignore operation currently not supported. Available in future release.

&#x20;

![](.gitbook/assets/image\(1162\).png)

&#x20;

Next is the validation screen.

If there is anything wrong with the data, it will be shown after validations are run.

&#x20;

![](.gitbook/assets/image\(1163\).png)

&#x20;

Once validated and no errors occur, select Next and then Import.&#x20;

&#x20;

![](.gitbook/assets/image\(1164\).png)

![](.gitbook/assets/image\(1165\).png)

&#x20;

The structure has been successfully loaded into Convergence PIM.&#x20;

Next, it's time to load [Parts.](how_to_import_new_classification_data.md)&#x20;
