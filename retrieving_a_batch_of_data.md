**Retrieving 
a Batch of Data**


 


**To retrieve a batch of data loaded into a terminal and display it 
in the Data Developer:**


 


·      Select 
the **Data Developer Module** from the Application Toolbox


 


·      Note that 
there are several **Filter** fields at the top of the screen. Setting the 
Filter fields prior to retrieving data will limit the number of items listed 
and simplify the data development task


 


·      Press the 
down arrow to the right of the **Batch** field and select the batch name for 
the data to be reviewed


 


·      Note that 
the User field lists the current user. This filter specifies which items to 
retrieve based on the assigned user. If desired, change the current, logged-in 
user name


 


·      Some 
items in the batch may be developed before others and their status field will 
be updated. The Status field may be set to retrieve only those items with a 
particular status. Press the down arrow to the right of the **Status** field 
and select the status


 


·      Move the 
curser through the classification tree, opening categories until the target 
terminal node is displayed


 


·      Select 
the target terminal node.  To ensure 
that all attribute data is displayed, be sure to select a terminal node. See 
the procedure below for retrieving data across multiple categories.


 


·      Note that 
the **Retrieve** button becomes available


 


·      Press the 
**Retrieve** button to display the data that matches the filter criteria


 


·      A batch 
of data can be retrieved, with some restrictions, when the items have been 
assigned to multiple categories. These restrictions also apply to Working 
Batches defined in the Item Browser and used to retrieve data from multiple 
categories.


 


 


**To retrieve a batch across multiple categories:**


 


- Select the **Data Developer Module** 
 from the Application Toolbox

 


- Press the down arrow to the right of the 
 **Batch** field and select the batch name for the data to be reviewed

 


- Set the filter values for the **User** and 
 **Status** fields

 


- Move the curser through the 
 classification tree, opening categories until the target category is 
 displayed

 


In the previous section, the 
target was a terminal node. In this procedure, a parent category may be 
selected. 


 


·                     
Select the target category. Note that the **Retrieve** button 
becomes available


 


·                     
Press the **Retrieve** button to display the data that matches 
the filter criteria. 


 


Restrictions for retrieving data across multiple categories 
include:


·         
Only attributes that are common to all subordinate categories will 
be displayed in the Data Developer spreadsheet



·         
Inheritance must be set for all subordinate categories, or the 
operation will not retrieve any items 


·         
Allowed Values lists cannot be accessed when retrieving from 
multiple categories. These values can still be modified manually.
