**Approving Category Attributes**



Gang 
approval of category attributes can be achieved in the classification manager.  
For those who have category approver rights, using the right click menu, users 
will be able to gang approve category attributes.  Users can gang approve 
category attributes for just the cateogry they are working on or they can also 
cascade those changes to other inhertiated categories below.  When the option 
to cascade approvals is availabe, a dialog box will appear describing how your 
status changes will be applied.


 


**Guidelines 
to consider when gang approving category attributes:**



·         
If 
you’re selecting a high level category in your structure, it may take a minute 
for the right click menu to appear.


·         
To 
approve a category attribute, the master attribute needs to be approved in 
attribute manager (standard DFR approval workflow).  If it’s not approved you 
will notice the submit menu will be grayed out.


·         
Sort 
category attributes by status and gang approve attributes of the same status.  
This is not required but may make for a better workflow.


·         
If 
the category attribute status is pending; make sure user’s select accept versus 
submit.  Only accept will take the status to approved.


 


Cascading attribute approvals may not always 
work.  If for example one of the category attributes is already approved at a 
lower inheritated category then the right click menu will not allow a user to 
change the status of this attribute.  When this happens **all of the approval 
functions are grayed out** in the right click menu options (see below).  
There is a way to override this condition if you are a category administrator.  
Category administrators can use control keys (control-shift) in combination 
with right click and the approval options will not be grayed out.


 


**Control-shift Override:**


 


A category administrator, using the control and 
shift keys in combination with the right click menu will allow a user to 
approve an inheritated attribute that may have different statuses. The control 
shift over-ride feature needs to be enabled in the options manager to properly 
work. There are other cases where control-shift over-ride will not work, it’s 
due to the fact that the master attribute has not been approved in the 
attribute manager.
