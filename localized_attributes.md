# Localized Attributes


Definition:


Adding localized attributes provides users the ability to duplicate an existing attribute with a different culture (language) by creating 2 different attributes.


 


 




<iframe src="https://www.youtube.com/embed/jNUeeLpJPEY/?wmode=transparent" width="560" height="315" frameborder="0" allowfullscreen></iframe>


 


 


• Localized attributes keep the same definition except:


- Culture: preferred language
- Description: attribute description is unique to the attribute
- Name: attribute name is unique to the attribute


• Localized attributes are assigned to all same categories


 


In the Attributes page, select the desired attribute; the details for the selected attribute will display on the right side of the screen.


Select the localized attribute section.


Once expanded, select 'Add'


![](assets/image(882).png)




 


Select the desired culture (language).


Enter the name and description in the culture (language) selected.


Select 'Add'.


![](assets/image(935).png)




 


Once the localized attribute has been added, notice the link icon displays next to the attribute on the left side. 


Also, the localized attribute name, description, and culture (language) display in the details on the right side of the screen.


![](assets/image(936).png)




 


Now the localized attribute has the same properties as the original attribute. 


![](assets/image(937).png)




 


If you change any of the properties to the original or localized attribute, the same changes will be done to the original or localized attribute.


In this example, data type is changed from String to Integer.


![](assets/image(938).png)




 


These new changes are reflected in the linked attribute.


![](assets/image(939).png)
