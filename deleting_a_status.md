# Deleting a Status

Deleting\_a\_Status - Design For Retrieval (DFR) Help

## Deleting a Status

Deleting a status is done when a status is no longer necessary and will remove the status indefinitely.&#x20;

&#x20;

Permissions need to be set and allowed for users to utilize the catalog manager and status manager.&#x20;

These permissions include:&#x20;

-Classification Permission: Category Permission/ Edit Items

-Catalog Permission: Access Catalog Status Module &#x20;

Deleting a status is easily accomplished by navigating to the \*\*[Status Mangement](deleting_a_status.md)\*\*screen, selecting the status and then " Delete" on the status to the left.&#x20;

&#x20;

![](.gitbook/assets/image\(940\).png)

&#x20;

A status can only be deleted as long as it is not tied to any categories, attributes or parts.  &#x20;

If a status is tied to any of the three elements listed above, an error message will be prompted:

&#x20;

![](.gitbook/assets/image\(941\).png)

&#x20;

As long as the status is not tied to any of the elements listed above, a window will be prompted seen below:&#x20;

&#x20;

![](.gitbook/assets/image\(942\).png)

&#x20;

Select Yes. The status is now deleted.&#x20;
