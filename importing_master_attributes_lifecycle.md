# Importing Master Attributes


 


If you would like to import Master Attributes using the Import Manager, you must first create your Exel import file. 


 


1. You must name the column headers exactly the same as described here or else DFR will not auto-map the different fields. 


 


Name


Description


Data Type


Status


Data Type


 


You can name the Excel file whatever you would like but the sheet must be named "Attributes". See the image below. 


 


![](https://userfiles-kb.s3.amazonaws.com/userfiles/13115/16830/ckfinder/images/qu/7738/image.png)




 


 


You can now import this file into DFR using the Import Manager located in the Thick Client (Desktop Application). 


 


 


2. Open the Thick client and navigate to the Import Manager, we will be doing a Classification Import. 


 


![](https://userfiles-kb.s3.amazonaws.com/userfiles/13115/16830/ckfinder/images/qu/3802/image.png)




 


 


3. Click on the Browse button and navigate to the Excel file you just created. Note: You cannot have the Excel file open in the background while importing. Make sure it is saved and closed before you choose it in this step. 


 


![](https://userfiles-kb.s3.amazonaws.com/userfiles/13115/16830/ckfinder/images/qu/519/image.png)




 


 


4. In this screen choose Excel as the import type. 


 


![](https://userfiles-kb.s3.amazonaws.com/userfiles/13115/16830/ckfinder/images/qu/8395/image.png)




 


 


5.  Because we are importing master attributes into DFR uncheck all of the boxes except for "Import Attribute Data". 


 


![](https://userfiles-kb.s3.amazonaws.com/userfiles/13115/16830/ckfinder/images/qu/191/image.png)




 


 


6. If you created your Excel file correctly as stated in step 1, all of the fields should be automapped. If not, then you can click the drop down menu and choose which mapping you would like. 


          If you are creating brand new attributes that do not exist in DFR you must choose the create operator. If you are updated existing then you can choose the Update operator. 


 


![](https://userfiles-kb.s3.amazonaws.com/userfiles/13115/16830/ckfinder/images/qu/9866/image.png)




 


7. Next click Validate and if there are any errors DFR will tell you here, if not then you are all set to import. Click the import button and your new attributes will be created or updated. 


 


8. To complete the process we can verify that our new or updated attributes are in DFR Online and available to use with our item data. Navigate to SmartClass and click the attribute button on the left hand menu, you can then search for the attribute you just made and it will show up in the list.


 


![](https://userfiles-kb.s3.amazonaws.com/userfiles/13115/16830/ckfinder/images/qu/7365/image.png)
