**Navigating the Classification Tree**


 


The center column of 
the Classification Manager tool lists all of the categories defined to capture 
product and part data, and the hierarchy in which this data will be 
entered. 


 


![](/userfiles/13115/16830/ckfinder/images/import-cbg4qpn20hv5/CM-Navigating_the_Classification_Tree_files/image001.jpg)




 


The **Folder** icon 
represents a **category** or **branch** in the hierarchy. The highest 
levels of categories in a Classification are called the super-categories. Lower 
categories are referred to as **sub-categories**, **branches** or **nodes**.


 


A **Plus Sign** next 
to the folder icon indicates that it is a **Parent** node and further 
branches, or **Children**, have been identified under this branch. Clicking 
on the Plus Sign will open the folder and display the next subordinate level of 
folders or terminal nodes.


 


A **Minus Sign** 
indicates the lowest level category. This category only has **Terminal Nodes** 
below it. Terminal nodes are represented by a page icon. A terminal node can be 
referred to as a leaf and is the repository for individual items. Each item 
assigned to a terminal node  is represented by a unique identifier and 
varies only in the values assigned to its attributes. These individual items 
should only appear one time in an enterprise-wide database. Clicking on the 
Minus Sign will close the folder and collapse that branch of the catergory 
tree.


 


The color of the folder 
or page icon to the left of the category or terminal node name indicates an **approval 
status** designating the [Status](javascript:popuplink('CM-Category_Statuses_With_Change.htm')) of the node. 


 


Selecting a category 
name will display the **Category Definition** in the right column of the 
Classification Manager tool.
