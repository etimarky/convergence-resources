# Creating a New Status


Creating a new status must take into consideration what the status is going to encompass: parts, categories, attributes or any combination of the three. 


 


The user must also take into consideration how many statuses between New and Pending that they'd like their team to follow. 


New, Pending, Approved and Rejected cannot be edited or deleted. 
Permissions need to be set and allowed for users to utilize the catalog manager and status manager.   

These permissions include:   

-Classification Permission: Category Permission/ Edit Items  

-Catalog Permission: Access Catalog Status Module
Navigate to [**Status Management**](#) under the **[Catalog Administration](#)** screen.   

 


![](assets/image(896).png)




  

To create a new status, select "Create." 


  

![](assets/image(897).png)




 


Type in a name for the status and determine what items that the status will cover.  


 


Check boxes are available for Categories, Attributes and Parts. The statuses will only show up on the items that are checked.  

One or multiple boxes can be checked on each status: this means the status will apply to that selection.   

  

Example 1:  If Categories is the only checkbox selected then when the user is changing an attribute status, the status would not show up as one of the selection.   

Example 2:  A category and part can both be selected to go through "New, In Work, and Classification" whereas an attribute may only need to go through "New and In Work." 
 


![](assets/image(898).png)




The color of the status will be what the user sees when the status is changed. 


This will show up in both ThickClient and Convergence PIM Online. 


 


![](assets/image(899).png)




 


Select Create, and the status is ready to be used.
