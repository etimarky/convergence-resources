



hs9000 - Design For Retrieval (DFR) Help




**User 
Manager**


 


The 
DFR User Manager provides user administration functions including: 1) Creating 
DFR user accounts, 2) Assigning roles for each DFR catalog, and 3) Describing 
the privileges each role offers.


 


Security 
mechanisms, referred to as “roles,” allow specific users to perform assigned 
actions while barring other non-privileged users from performing actions.  
There are catalog roles and category roles.  Catalog roles apply to the entire 
DFR application and its individual modules.  Category roles only apply to the 
Classification Manager.  Category roles define what access users have to a 
portion of the catalog tree.  (For more details, see the Classification Manager 
help section.  Every instance of DFR has just one DFR Administrator.  The 
administrator has the rights to create users and to assign users’ rights in any 
catalog.


 



**Description of DFR Roles:**


A 
role defines the activities that can be performed by a user.  There are 
currently 10 roles in DFR including: Allowed Values Manager, Attribute Authorizer, 
Attribute Developer, Catalog Administrator, Category Administrator, Exporter, Item 
Approver, Item Assignment, Item Loader, and Vendor.


 


**Allowed Values Manager -** The Allowed Values Manager role allows a user 
to create and edit allowed values list for attributes both at a global and 
local level.  This role can also edit attribute data by batch or across 
batches.



**Attribute 
Authorizer** 
- The Attribute Authorizer role allows a user to approve attributes in the 
Attribute Manager application.


 


**Attribute Developer -** The Attribute Developer role allows a user to 
create, edit and delete attributes in the Attribute Manager application.


 


**Catalog Administrator -** The Catalog Administrator can assign all 
roles for users for a particular catalog.   Catalog Admin cannot create or 
disable DFR user accounts.  Only the DFR Administrator can do this.  Catalog 
Administrators can also change the status of an item.


 


**Category Administrator -** The Category Administrator can assign 
category roles such as Category Developer and Category Approver in the Classification 
Manager.  Users with Category Developer rights are permitted to create and edit 
category properties whereas users with Category Approver are permitted to 
approve or reject proposed category changes.  These roles are assigned to users 
at a particular level in the classification structure and can be inherited to 
lower levels in the classification structure.  These roles are set in the 
Classification Manager, not in the User Manager since they can be specifically 
defined independently in the classification structure. (See Classification 
Manager help for more detail).  The Category Administrator can also add new 
attributes to approved categories and change attributes and have those changes 
propagate to approved categories.


 


Any 
user assigned as a Category Administrator can now make and approve changes to 
any category in the Classification Manager, regardless if their user id appears 
as a Category Approver or Developer.



**DFR Administrator -** The DFR Admin can assign all roles for users 
for any catalog and can also create and disable user accounts.  Only the DFR 
administrator can do this.  You have to log into DFR with the Admin user name, 
not your normal user account, to execute this role.  It’s not recommended a 
user make standard data changes while logged in as admin, since you may lose 
the audit trail capabilities to track changes.


 


**Exporter -** The Exporter role allows a user to export data from DFR 
to other systems.


 


**Item 
Approver** 
– This role allows user to change the status of an item.  Either the Catalog Administrator 
or Item Approver can change the status of an item.


 


**Item 
Assign** 
- The Item Assign role allows a user access to the Item Assignment application 
and assign user IDs to items.


 


**Item Loader -** The Item Loader role allows a user access to the Item 
Loader application.  Only one person can load data at a time.  Multiple users 
can be logged into the Item Loader.  If a user tries to execute a load while 
another user is in the process of loading, a warning message will appear.  This 
same condition can occur if someone is using the Item Key Changer (Reset Org or 
Rename Item) while another user is loading.


 


**Vendor** – This role only 
allows this DFR user to work in the data developer and they can only see the 
items assigned to them to work on.  The Vendor role does not have access to the 
other DFR modules. It is a highly restricted role in DFR.




**Adding a User to a Catalog:**


Only 
DFR Administrators or Catalog Administrators can add users to a catalog and 
assign rights or roles for the user.  


 


To 
add a user to a catalog, follow these steps:


 


1.)   Select the catalog 
from the list of catalogs


2.)   Select edit to view 
and edit the catalog properties, including users


3.)   Select a user from 
the list of users


4.)   Select the single 
arrow pointing to the members list


5.)   The user should now 
be added to the list of members


6.)   Select save


 


The 
new user will now be able to access the selected catalog.


 


 


**Assigning Catalog Roles:**


To 
assign a role to a user for a specific catalog, follow these steps:


 


1.)   Open the folder of 
the catalog where you want to assign the role


2.)   Select a user from 
the list of members


3.)   Now check any of the 
roles you want to apply to the user


4.)   Select save and the 
user roles will be updated  
  
  



A 
Catalog Administrator or DFR Administrator can only perform these actions.


 


**Creating a User:**


Creating 
DFR User Accounts for gaining access to DFR catalogs is done with the User 
Manager application only by the DFR Administrator. The User section of the User 
Manager allows an Admin to set up new users or change account information for 
particular users, including user name and passwords.  Selecting the Users 
Folder will provide a list of users and the rights that are assigned for the 
current database.  The DFR Administrator can add users from this window and 
disable user accounts.
