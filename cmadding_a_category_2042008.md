**Adding a Category**


 


**The category developer role for the parent category is required to 
be able to add a subordinate category.**


 


To add a category:


·         Highlight the parent category by placing the cursor on the category 
and pressing the left mouse button.


·         Press the right mouse button to display the **Category Operations** 
drop down menu.


·         Select **Add Category**.The Category Maintenance window 
is displayed. Note that the path to the new category is displayed in the **Parent**field at the top of the window.


·         Enter the name of the new category in the [**Name**](javascript:popuplink('hs2160.htm')) field. This is a required field.


·         Enter the description of the new category in the[**Description**](javascript:popuplink('hs2170.htm')) field.  This is a required 
field.


·         [**Properties**](javascript:popuplink('hs2750.htm')) and **Property Values** may be added to the category at this 
time.


·         Press the down arrow to the right of the **Approval Type** 
field and select the Approval process for this category


·         Press the down arrow to the right of the [**Status**](javascript:popuplink('CM-Category_Statuses_With_Change.htm')) field and select the Status for this category


·         Determine whether this category should [**Inherit Attributes**](javascript:popuplink('hs920.htm')) from the parent category by selecting the check box


·         Determine whether this category should [**Inherit Security**](javascript:popuplink('hs920.htm')) from the parent category by selecting the check box


·         Note that the status of the **Terminal Node** field is **Yes** 
and cannot be changed. The system will automatically set this field to Yes 
until a child category is created associated with this node


·         
When 
the information about the node is complete, press the **Update** button at 
the bottom of the Category Maintenance window. If information is not entered 
into the Name or Description field, a red dot will flash next to the missing 
information.


Enter the missing 
data, and press **Update** again.


When adding a child category to a 
terminal node which contains items, a dialog will prompt the user explaining all 
items in the current terminal node will be moved to the new category 
created.  


![](/userfiles/13115/16830/ckfinder/images/import-d2f34nel28gg/CM-Adding_A_Category_files/image001.png)




Selecting **Yes** will display 
the Category Maintenance window.  Once Update is clicked the items from 
the now parent node will automatically be moved to the newly created child/terminal 
node.
