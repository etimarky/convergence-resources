An alias property allows a category developer to assign an 
alternative name to an attribute label. For example, "ItemDescr 
Alias" would allow the category developer to specify an alternate label 
for the Item Description column in Data Developer, Data Validator, SmartFind, 
and WebDFR.


 


**Assigning Values to a Category Property**


 


Properties on categories apply to the entire catalog. However, 
specific values can be assigned to a category by the category developer. The 
assigned value will apply to the current category and all subordinate 
categories with **Inherit Attributes** checked.


 


**To view category properties:**


- Open the 
 classification tree and select the target category
- Select the **Properties** tab in the category description to view properties currently assigned 
 to the category
- Right click on the 
 category
- Select the **Update 
 Category...** action

The Category maintenance window will be displayed.


 


**To assign a value to a category property:**


- Select the target 
 category and right click
- Select the **Update 
 Category...** action

The Category maintenance window will be displayed


- Place the cursor 
 in the **Value** field for the target property
- Type in the value
- Press the **Update** button

Note that the value for the property has been updated in the 
Property Value column.


 


**Hiding System Attributes:**


 


To hide system attributes go to one of the alias category 
properties and follow these steps.


- Delete the **<EMPTY>** 
 label or any other value set from the **Value** column
- Press the **Update** button

Notice that the property has been added to the list of properties 
assigned to the category


 


**To reset a property value to <EMPTY>:**


- From the Category 
 Maintenance window, select the **Properties** tab
- Right click on the 
 property value to be changed
- Notice that a 
 Reset button is displayed to the right of the cursor
- Select the **Reset** button

The field will be reset to **<EMPTY>**


 


**NOTE:** Leaving the property Value field blank will cause the property 
value to be blank and will hide the attribute from view in Data Developer, Data 
Validator, SmartFind, and WebDFR.


 


**To cascade property value down tree:**



In certain circumstances, you may want to force a category 
property to be inherited all the way down the tree regardless of what has been 
set at the lower levels, effectively overriding all set values.  To do this, 
select the **Cascade Property Changes To All Descendants** option located 
directly underneath the property grid.
