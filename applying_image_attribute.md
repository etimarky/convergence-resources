# Applying Image Attribute

Applying\_Image\_Attribute - Design For Retrieval (DFR) Help

## Applying Image Attribute

Go into Convergence PIM and select SmartClass.

&#x20;

![](.gitbook/assets/image\(722\).png)

&#x20;

Select the Attributes icon to create an attribute.

&#x20;

### ![](.gitbook/assets/image\(808\).png)

&#x20;

Update the Description, and update the Data Type to "File".&#x20;

&#x20;

![](.gitbook/assets/image\(809\).png)

###

Also, update all the properties under this attribute like following changes below:

* File Type: Select "Image"
* Classification Category: Select the File Category that was created earlier. Images will be stored in that category
* File Storage: Select "Azure Storage"
* File Storage Configuration: Select "Default" or Select "Category Images"; this depends on the Storage Configuration

![](.gitbook/assets/image\(810\).png)

&#x20;

Go to the Parent or child categories where images will be displayed and add the created "Image" attribute.&#x20;

&#x20;

![](.gitbook/assets/image\(812\).png)

&#x20;

Double-click on the attribute that was just added and then click on Edit.

![](.gitbook/assets/image\(813\).png)

&#x20;

Now update the Summary Attribute to be "True" under Properties.&#x20;

&#x20;

![](.gitbook/assets/image\(814\).png)
