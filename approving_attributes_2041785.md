



Approving\_Attributes - Design For Retrieval (DFR) Help




**Approving 
Attributes**


 


 


Attributes can only be set to approved by the user with Attribute 
Approver role.  Attribute Developers can set other statuses but they can 
not set attributes to an approved status.  Attributes can be approved one 
at a time or they can be approved in groups.  Attributes have to be 
approved before they can be approved for use on a category in the 
classification manager as part of the standard DFR attribute and category 
approval workflow.


 


 **Approving an Attribute: (Individual Approval)**


 


·         
Select 
the attribute to approve and right click


·         
Attributes 
can be set to Pending, Approved, Rejected, or any User Defined status 
applicable to Attributes. User Defined statuses appear in a sub-menu under 
Rework.


·         
Select 
one of the available approval options


·         Review the 
approval change from the status field located top right of the module


 


**Approving 
Multiple Attributes Simultaneously: (Gang Approval)**


 


·         
It’s 
recommended to use the status filter so that all the attributes are of the same 
status before you start.


·         
Use 
the shift key to select multiple attribute in sequence.  Use the control 
key to select multiple attributes not in sequence.


·         
Now 
right click to select the approval option, select one of the available options 
not grayed out.


·         Review the 
approval change from the status field located top right of the module to 
confirm the attribute status changed.
