**Updating a Category**


 


This operation is used to modify an existing 
category. For complete information about the fields in the Category Maintenance 
window, refer to the [**Adding a Category**](CM-Adding_A_Category.md) topic.


 


Fields that can be modified, and the allowed 
modifications include:


 


- Category 
 Name and Description - these fields can be modified
- [Properties](CM-CategoryProperties.md) - Properties can 
 be assigned values, or existing values may be modified
- [Attachments](javascript:popuplink('hs1902.htm')) may be added and 
 the attachment fields may be defined
- The 
 Approval or [Authorization Type](javascript:popuplink('hs903.htm')) may be modified
- The 
 [Category Status](javascript:popuplink('CM-Category_Statuses_With_Change.htm')) may be changed
- [Inheritance](javascript:popuplink('hs920.htm')) of attributes and 
 security for the category may be modified
- [Editing Attribute 
 information](javascript:popuplink('hs1360.htm')) 
 can allow an attribute to be customized for a category

 


**The category developer role for the parent 
category is required to be able to update a subordinate category.**


 


**To update a category:**


 


- Press 
 the right mouse button to display the **Category Operations** drop down 
 menu.

 


- Select 
 **Update Category...**.The Category Maintenance window is 
 displayed. Note that the path to the new category is displayed in the **Parent** field at the top of the window.

 


- Enter 
 the updates to the desired fields.

 


- When 
 the information about the node is complete, press the **Update** button 
 at the bottom of the Category Maintenance window.

 


Return 
to [How to Build a Classification](CM-HowtoBuildaClassification.md).
