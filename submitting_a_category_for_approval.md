**Submitting a Category for Approval**


 


 


When a Developer creates a new category and 
defines its characteristics, the new node must be submitted for approval. An 
Approver reviews the submitted category, and either approves it, rejects it, or 
sets it to "rework" by assigning the status to one of the User 
Defined statuses. Once the category is defined appropriately, it is approved.


 


Attribute assignment to a category must also be 
approved. When a category has been submitted for approval, and its status is Pending, 
the category approver reviews each attribute assignment. The attributes 
assigned to the category are approved, and then the category is approved. Once 
it is approved, no additional attributes can be assigned. To make additional 
changes to an Approved category, first set the status to one of the User 
Defined statuses, make the necessary changes, and resubmitted for approval.
