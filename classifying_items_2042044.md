



Classifying\_Items - Design For Retrieval (DFR) Help




**Classifying 
Items**


 


The Data Developer allows a user to classify items into the 
correct category in the classification tree. Only those items assigned to the 
current user can be classified by that user.  
 
 


 


**To classify items:**


 


- Retrieve the item or items you want to 
 retrieve from the terminal node (source category) and assign them to you 
 if they are not already.
- Open the classification tree until the 
 destination category is displayed.
- Use the shift key to select a group of 
 item(s) or the control key to select individual item (s).  Make sure to 
 select the field in the first column and not other attribute fields.  When 
 selecting multiple, non-sequential items, use the control (Ctrl) key vs. 
 the shift key.  The shift key is good to use when the items are listed in sequential 
 order.
- Once the items have been selected, release 
 the shift or control key yet key your mouse key down, then move selected 
 item(s) to the destination category. A small rectangle will appear under 
 the cursor indicating that the cursor is over a particular category.
- A message will appear displaying the 
 number of items to be assigned from the source category and the user will 
 be asked to verify the destination category.
- If a user is moving data from one 
 category to another, where the source category has different attributes 
 than the destination category, the user will be warned that certain 
 attribute data maybe lost during classification.  If the source category 
 has a subset of attributes to the destination category, then there will be 
 no warning.
