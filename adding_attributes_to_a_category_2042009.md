**Adding Attachments to a Category**


 


Attachments provide a way to associate documents with a category 
or node.  All the items associated with the node will have a link to the 
documents that are listed as attachments.


 


In order to use attachments, the Classification must have a 
category with the path **Root\Non-Parts\Attachments.**



The Attachments category must have an attribute called “URL” set 
up as a string data type.  The URL for this attachment must then be mapped to 
this attribute.  


 


To assign or remove attachments, the category must not have an 
Approved status.  The user must have the Category Developer or Category 
Approver role for the category where the attachment is being assigned.  The 
attachments do not have to be assigned to the user making the assignment.  


 


**Using the Classification Manager:**


·            
Select the category where the attachments will be assigned


·            
Right click on the category name and select **Update** category


·            
The Category Maintenance window is displayed


·            
Select the Attachmentstab


·            
Manually type in the Attachment Number and the Local Qualifer


·            
Select the **Update** button to save the changes and the Item 
Description and URL fields will automatically be updated


 


**To view attachments** related to a category, open the Attachments 
tab for a category and click on the Attachment number, not the URL, to view the 
attachment.  All the items associated with the node will have a link to the 
documents that are listed as attachment.


 


**To delete an attachment** from a category:


Deleting an attachment from a category will delete the attachment 
from all subordinate categories.
